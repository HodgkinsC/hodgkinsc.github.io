Getting Started with Godot

These guides use Godot 4.5, this guide may not work with other versions of the engine.

This project will be a simple 2D platformer, the guide will go in-depth about everything you do and explain what you're actually doing.

So, opening Godot for the first time and every consecutive time after that will give you this screen, or something similar to it.

(It may look slightly off and that's because I had to modify the image to remove the projects I already have)

This is the Godot Project Selector. It is where you Select your Godot Projects. To create a new project, click create at the top right corner of the window and you'll see this pop up:

(If you're like me then you'll understand how annoying it is to name projects you haven't made yet, on my usual projects I usually just give them random names that are vagely similar to what I want to do. Some of my professional projects have names like "codering". For this I'm just calling it "Website Guide" but the project name doesn't really have any bearing on the actual project and it can be renamed later.)

After setting the project name, set the project path to a place that's useful and reasonable, it's good to create a "Projects" folder and add the projects in there. If you have the "Create Folder" check box checked then it'll create a new folder with the project name inside the folder path specified below it.

The next section below the path isn't really relevant for this guide but I'll explain it anyways. The Renderer tab lets you select which renderer the project will use (This can be changed later.) If you select them then it'll show what the renderer is good for.
The "Forward+" renderer can only run on desktop platforms, not on mobile or embedded in websites.
The "Mobile" renderer is less optimised for more advanced scenes but it can be run on both Desktop and Mobile devices, not embedded in websites though.
The "Compatibility" renderer is optimised for much simpler scenes and can be run on Desktop, Mobile, and be embedded into websites.

This project will be simple and I intend to place the finished project in this website, so I'll use the "Compatibility" renderer.

The final section is the "Version Control Metadata". This section is for people who use version control programs like "Git". We won't be using version control for this guide so set it to "None"

(Also check "Edit Now" if it isn't already.)

And with the settings set, you can now press "Create".

After waiting for the project to load, you will be met with a view similar to this. You'll probably be automatically placed into the 3D view, but this project will be 2D so switch to the 2D tab at the top centre of the screen.

First, starting on the left side of the screen at the top, there's the Scene Hierarchy view. It displays every game object that's included in the current scene
Underneath it is the File System view, it shows you the inside of the project folder, this is where all game files will be stored (Excluding save data, which we will not get in to). There should already be a single file in the folder: "icon.svg". It's the image used for the project's icon.

On the right side of the screen is the inspector, when you have a game object selected, it will display all information about that game object, like it's postition, rotation and scale for example.

You may have noticed that there are more tabs at the top of each of the mini windows, we'll get into those later.

Since we are making a 2D game, you'll want to select "2D Scene" under Create Root Node in the Scene Hierarchy
